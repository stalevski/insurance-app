How verdicts were assigned. The platform is both a QA testing ground and a nascent deployable product. Features were judged on (a) fit with the existing SourceContracts → Mappers → Canonical → Flows → Responses architecture, (b) whether they complete an already-advertised capability, and (c) value vs. effort. “Skip” means out of scope or better solved by infrastructure, not “bad idea”.
